Проект представляет собой pet-framework для автоматизированного тестирования
REST API и Web UI на примере системы управления тест-кейсами (Qase).

Цель проекта:
- показать архитектурное мышление
- продемонстрировать навыки API и UI автоматизации

## Technology Stack

### Core
- Java
- TestNG
- Selenium / Selenide
- Lombok
- REST Assured
- Jackson
- Gson

### Reports and logs
- Allure
- Log4j2

### CI
- GitHub Actions
- Maven

##  Test Coverage
###  API Tests
#### Projects
Реализованы API автотесты, покрывающие основные сценарии работы с проектами:
- создание проекта и получение проекта по `code`
- проверка корректности возвращаемых полей (`title`, `code`)
- удаление проекта и проверка его недоступности после удаления
- валидация обязательных полей при создании проекта (`title`)
- проверка уникальности `project code` при создании проекта
#### Cases
Реализованы API автотесты для работы с тест-кейсами внутри проекта:
- создание тест-кейса в проекте (*smoke-тест*)
- обновление тест-кейса методом `PATCH` (изменение поля `type`)
- удаление тест-кейса и проверка его недоступности
- добавление нескольких тест-кейсов и проверка корректного обновления счётчиков проекта
- фильтрация тест-кейсов по типу (`SMOKE` / `REGRESSION`)
###  UI Tests
#### Projects
- **Создание проекта через UI** (*smoke-сценарий*)  
  Проверка успешного создания проекта через веб-интерфейс.
- **Удаление проекта**  
  Проверка возможности удаления ранее созданного проекта.
- **Негативный сценарий: создание проекта без названия**  
  Проверка, что проект **не создаётся** при отсутствии обязательного поля `Name`  
  и срабатывает браузерная HTML5-валидация формы.
#### Cases
- **Создание тест-кейса в проекте через UI** (*smoke-сценарий*)  
  Проверка создания тест-кейса и корректного увеличения их количества в проекте.
- **Создание тест-кейсов с различными входными данными**  
  Параметризованный тест с проверкой сохранения всех полей тест-кейса.
- **Проверка сохранённых данных тест-кейса**  
  Открытие формы редактирования и сравнение данных с исходными входными параметрами.
#### Suites
- **Создание тест-сьюта через UI**  
  Проверка возможности создания тест-сьюта и его отображения в интерфейсе проекта.
